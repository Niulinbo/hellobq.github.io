js-Array相关API

发表于 2018-08-07 | 分类于 javascript |

数组常用属性

  • .constructor 返回对 Array 构造函数的引用。
  • .length 返回当前数组的长度。

数组常用的方法

  • arrName[index] 通过索引访问项。当访问的索引 >= 数组长度时，返回 undefined；当给索引 >= 数组长度的项赋值时，会改变原来数组的长度。
  • .push(n1[, n2]) 在数组尾部向数组插入 n1... 项，返回数组新长度。
  • .unshift(n1[, n2]) 作用同上，不过是在数组头部插入。
  • .pop() 从数组尾部删除某个元素。返回被删除一项。
  • .shift() 作用同上，不过是从数组头部删除一项。
  • .splice(index, n[, n1, n2]) 有删除、替换、增加功能。从索引为 index 位置开始删除 n 个，然后在索引 index 后增加 n1, n2.... 这些项。如果 n=0 ，就是不删除，只是往数组内增加项。如果n和后面选项个数相同，就是替换。如果没有后边的选项，就是简单的删除。返回删除项组成的数组。
  • .reverse() 将数组反转。返回反转后的数组。
  • .concat(arr[, n1, n2]) 数组合并返回新的数组。源数组未变。 注意：参数是一维数组还是单个项都会被解析出来。
  • .join([str]) 数组的项以某个字符串 str 连接成一个字符串。无参数，默认 str = ','
  • .forEach(fn) 遍历数组。无返回值。回调函数中的参数分别为：item 项，index 索引，array 当前数组。
  • .map(fn) 作用同上。返回一个由回调函数返回值组成的数组。注意：回调函数不写返回值，默认返回 undefined 的。
  • .every(fn) 遍历数组，用来判断数组每个元素是否都符合某个条件。如果回调中有一个返回 false，那么这个函数就返回 false。
  • .some(fn) 遍历数组，用来判断数组每个元素是否符合某个条件。如果回调返回值中有一个返回 true ，则这个函数就返回 true。
  • .filter(fn) 遍历数组。返回回调函数 fn 返回值为 true的项的集合，即返回符合条件的项组成的数组。
  • .reduce(fn) 为数组的每一项执行回掉函数。它很有用，主要是它第二个可选传参的使用。
  • .sort(fn) 没有回调则是按照 ASCII 码排序。有回调，则是对具体项的比较。

es6 对数组的新增

  • Array.of() 新增创建数组的静态方法，解决之前数组的创建受它参数影响的限制。
  • Array.from(arrName, fn) 将类数组转换成数组，回调函数中的参数及作用类似 forEach 的回调函数。返回一个数组。

  • .find(fn(value, index, array)) 找出第一个满足条件的项。
    它可以根据条件返回具体的项，而 indexOf 必须要实现知道这一项。另外，它也可以弥补 indexOf 的不足。

  • .findIndex(fn(value, index, array)) 找出第一个满足条件项的索引。

  • .fill(num) 返回被 num 填充的数组，num还可以是字符串、json 等数据格式。
  • ...arrName 扩展运算符，这样很容将数组内容扣出来。

需要注意的问题

  • 数组遍历除了以上原生 API以外，还可以使用 for...of ，因为数组的原型实现了 interator 接口。
  • forEach 遍历数组中途不能 break。可以使用 es6 中的 for...of，它可以途中 break。

数组的判定

  • Array.isArray(arrName) 不兼容 IE8-
  • arrName.constrcutor === Array 利用圆型链的特性。
  • ({}).toString.call(arrName) = '[object Array]' 利用数组原型继承了对象原型的 toString 方法。
  • 使用 instanceof 判断，在多窗口下会失效。
  • 使用数组独一无二 API：!!arr.sort !!arr.slice !!arr.join !!arr.splice 兼容 IE8-且简洁。推荐
