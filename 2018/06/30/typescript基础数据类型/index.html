typescript基础数据类型

发表于 2018-06-30 | 分类于 typescript |

typescript 类型注解

在 typescript 使用 : 类型 来标注变量具体的数据类型，对 javascript 变量随意性有了具体类型约束。

布尔值 boolean

数字 number

typescript 和 javascript 一样，所有数字都是以浮点数形式存储的，支持二、八、十、十六进制数字的变量。

字符串 string

任意值 any

使用 any 类型的变量目的：躲避掉编译时类型检查

数组 Array

typescript 使用两种方法定义数组。

  • 在元素类型后面紧跟着 []
  • 使用数组泛型 Array<元素类型>

元组 Tuple

如果一个数组的项的类型和个数都确定的话，可以使用元组来声明变量。

当访问已知索引的数据时，可使用其 api 来操作数据；访问一个越界的元素的，此时会使用联合类型 (string | number):

枚举 enum

枚举是对 javascript 数据类型的补充。意义为：物以类聚。为一组数据赋予一好听的名字。枚举成员不能有数值

空值 void

  • 只能给数据类型为 void 的变量赋值为 undefined | null；
  • 函数返回值类型为 void，实际返回值和 javascript 一样为 undefined；

undefined 和 null

默认情况下，这两种类型是其他类型的子类型。比如：可以使用 undefined|null 赋值给类型为 number 的变量。

Never

Never 表示永不存在值的类型。
never 类型是任何类型的子类型，也可以赋值给任何类型；然而，没有类型是 never 的子类型或可以赋值给 never 类型（除了 never 本身之外）。 即使 any 也不可以赋值给 never。(个人认为这个类型很鸡肋)

类型断言

可认为类型转换, 不进行特殊的数据检查和解构。它没有运行时的影响，只是在编译阶段起作用。
类型断言两种方式：

  • <数据类型>
  • as 推荐使用，当你在TypeScript里使用JSX时，只有as语法断言是被允许的。

注意
这种类型转换是能是大范围转换为小范围，例如：any => 其他类型，不能 number => string
